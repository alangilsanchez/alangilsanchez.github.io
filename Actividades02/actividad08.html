Introducción

SQL Injection (SQLi) sigue siendo una de las vulnerabilidades más críticas y comunes en aplicaciones web. Ocurre cuando un sistema permite que datos proporcionados por el usuario manipulen directamente la estructura de consultas SQL enviadas al motor de base de datos. En esta actividad se documenta la resolución práctica de los laboratorios oficiales de SQLi de PortSwigger Web Security Academy utilizando Burp Suite como herramienta de intercepción de tráfico.

Marco Teórico

Funcionamiento y Clasificación de Ataques

El ataque consiste en inyectar metacaracteres de SQL (como comillas simples ', dobles " o comentarios --) en parámetros vulnerables. Esto permite alterar la lógica lógica de la consulta original. Se clasifican principalmente en:

  • In-band (Classic): El atacante utiliza el mismo canal de comunicación tanto para lanzar el ataque como para recibir los datos (ej: UNION-based, Error-based).
  • Blind (Ciega): La aplicación no muestra datos ni errores en pantalla. Se debe inferir la información mediante respuestas booleanas (Boolean-based) o retardos de tiempo (Time-based).
  • Out-of-band (OOB): Se fuerza al servidor de base de datos a iniciar una conexión de red externa (ej: DNS o HTTP) hacia un servidor controlado por el atacante.

Mitigación Recomendada

  • Consultas Parametrizadas: Utilizar Prepared Statements (Sentencias Preparadas) que aseguran que el motor de base de datos trate la entrada del usuario estrictamente como datos, nunca como código ejecutable.
  • Validación de Entradas (Input Validation): Emplear listas blancas para filtrar caracteres no permitidos antes de procesarlos.
  • Principio de Mínimo Privilegio: Configurar el usuario de la base de datos con permisos estrictamente limitados (ej: solo lectura sobre tablas necesarias).

Conclusión

La resolución de estos laboratorios ilustra cómo técnicas desde una simple inyección basada en unión hasta inyecciones ciegas basadas en error o tiempo pueden llevar al compromiso total de un sistema, incluyendo la fuga de credenciales administrativas. La ciberseguridad defensiva debe enfocarse en la sanitización e inmutabilidad de las consultas como la primera y más efectiva barrera de defensa.
