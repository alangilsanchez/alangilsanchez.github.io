Resumen Ejecutivo

Este informe documenta la prueba de penetración técnica realizada sobre la máquina vulnerable My File Server 1. El objetivo consistió en identificar configuraciones erróneas, servicios expuestos y vulnerabilidades a nivel de kernel que permitieran a un atacante local o remoto obtener acceso administrativo (root). Se logró el compromiso total del sistema tras extraer credenciales mediante Samba y aplicar el exploit Dirty COW localmente.

Fases de la Auditoría

1. Reconocimiento y Enumeración

Se identificó la dirección IP del objetivo dentro del segmento local mediante escaneo ARP y se realizó un análisis profundo con Nmap. Los puertos expuestos descubiertos incluyen:

  • Puerto 21/TCP (FTP): Servicio de transferencia de archivos desactualizado.
  • Puerto 22/TCP (SSH): Permite accesos remotos seguros.
  • Puerto 80/TCP (HTTP): Servidor web Apache 2.4.6.
  • Puertos 139 y 445/TCP (Samba/SMB): Recurso compartido expuesto.

2. Explotación y Acceso Inicial

Mediante el uso de herramientas como smbmap y smbclient, se detectó un recurso compartido denominado smbdata accesible de manera anónima. Al enumerar su contenido, se halló un archivo readme.txt que exponía credenciales en texto plano para el servicio SSH. Con estas llaves de acceso, se estableció una sesión remota interactiva en el servidor.

3. Escalada de Privilegios Local

Una vez dentro del sistema como usuario regular, se analizó el kernel de Linux subyacente. Se detectó una versión vulnerable a Dirty COW (CVE-2016-5195), un fallo crítico de condición de carrera en el subsistema de memoria (Copy-On-Write). Se compiló localmente el exploit correspondiente y se ejecutó con éxito, sobrescribiendo el archivo /etc/passwd para inyectar una cuenta con UID 0 (root) y obteniendo acceso total como superusuario.

Tabla de Hallazgos Críticos

ID Hallazgo Vulnerabilidad Detectada Severidad Impacto Mitigación Recomendada
H-01 Recurso compartido SMB sin autenticar Alta Fuga de credenciales críticas. Restringir el acceso anónimo a recursos de red y aplicar cifrado.
H-02 Versión de Kernel vulnerable a Dirty COW Crítica Escalada total de privilegios a root. Actualizar y parchear inmediatamente el kernel del sistema operativo.
H-03 Uso de FTP no cifrado Media Interceptación de credenciales en tránsito. Deshabilitar el servicio FTP y sustituirlo por protocolos seguros (SFTP/SCP).

Análisis de Impacto (Modelo CIA)

  • Confidencialidad: CRÍTICA. El atacante obtuvo acceso a archivos de configuración internos y contraseñas guardadas en texto plano.
  • Integridad: CRÍTICA. La explotación de Dirty COW posibilitó la escritura en archivos protegidos del sistema, alterando la base de datos de usuarios.
  • Disponibilidad: CRÍTICA. Con privilegios de administrador total (root), un atacante podría destruir servicios web o inhabilitar el hardware.

Conclusión

El éxito de la intrusión resalta el riesgo inherente de mantener credenciales legibles en recursos compartidos y la falta de un ciclo de parcheo continuo sobre el kernel de los sistemas operativos. La remediación no solo debe centrarse en blindar el perímetro, sino también en asegurar que los sistemas internos no sean susceptibles a exploits locales conocidos de escalada.
