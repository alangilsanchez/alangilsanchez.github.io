1. Introducción

Las organizaciones enfrentan amenazas cibernéticas que comprometen la seguridad de sus sistemas. En esta actividad se aplica el Modelo Diamante de Análisis de Intrusiones para relacionar elementos clave como el adversario, la víctima, la infraestructura y las capacidades. Además, se complementa con la Cyber Kill Chain para comprender las fases del ataque.

2. Comprensión del Modelo

Elemento Descripción Ejemplo práctico
Adversario Actor responsable de utilizar una capacidad contra una víctima. Grupo de ciberdelincuentes con campaña de ransomware contra hospitales.
Capacidad Herramientas, técnicas y tácticas del adversario. Malware tipo troyano diseñado para evadir antivirus.
Infraestructura Medio físico o lógico para establecer contacto. Servidor de Comando y Control (C2) en la nube.
Víctima Objetivo del ataque. Departamento de finanzas recibiendo phishing.

3. Análisis de Evento

Escenario: Una organización detecta comportamiento anómalo. Se detecta malware, dominios C2, resolución a IPs externas, y múltiples hosts conectándose. WHOIS revela el posible origen.

  • Fase: Comando y Control (C2). Post-explotación.
  • Resultado: Compromiso activo de red.
  • Dirección: Saliente.
  • Metodología y Recursos: Malware emplea dominios específicos. Se usan IPs externas, y herramientas como WHOIS.

Preguntas Clave:

  • Adversario: Actor externo responsable de controlar el malware y dominios C2.
  • Capacidad: Software malicioso con función de ejecutar consultas a C2.
  • Infraestructura: Nombres de dominio e IPs externas (servidores remotos).
  • Víctima Primaria: Host inicial infectado (y la organización en general).
  • Movimiento Lateral: Sí, los logs muestran múltiples hosts conectándose a las mismas IPs externas.

4. Relación con la Kill Chain

Evento Fase Kill Chain
Detección de malware Instalación
Envío de phishing Entrega (Delivery)
Conexión a C2 Comando y Control (C2)
Ejecución del malware Explotación / Ejecución

5. Hilos de Actividad

Hilo 1:

  • Phishing dirigido a administrador → abre enlace → malware se ejecuta → conexión C2 → acceso remoto obtenido.

Hilo 2:

  • Uso de equipo comprometido como proxy → conexiones a otros dispositivos internos → segundo equipo comprometido → ampliación del alcance.

Relación: El atacante utiliza la primera víctima como punto de apoyo para desplazarse lateralmente (pivoting).

6. Conclusión

El Modelo Diamante es una metodología efectiva para identificar y correlacionar los elementos involucrados en un incidente. La integración con la Cyber Kill Chain facilita la comprensión de las fases del ataque, fortaleciendo el análisis forense, la detección de patrones y apoyando la toma de decisiones para mejorar las estrategias de protección tecnológica.
