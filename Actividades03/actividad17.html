1. Introducción

El objetivo de esta actividad es aplicar el estándar internacional CVSS v3.1 (Common Vulnerability Scoring System) para cuantificar la severidad de una vulnerabilidad detectada en un entorno web interno. Este proceso permite a las organizaciones priorizar la mitigación de riesgos basándose en datos técnicos objetivos y medibles.

2. Escenario de Evaluación

Una organización detecta una vulnerabilidad en un sistema web interno con las siguientes características:

  • Puede explotarse a través de la red.
  • La explotación requiere baja complejidad.
  • Se necesitan privilegios elevados.
  • No requiere interacción del usuario.
  • No cambia el alcance.
  • El impacto en Confidencialidad, Integridad y Disponibilidad es Bajo.

3. Desarrollo Técnico

Basado en los parámetros del escenario, se generó el siguiente Vector String que resume de forma técnica todas las métricas base: CVSS:3.1/AV:N/AC:L/PR:H/UI:N/S:U/C:L/I:L/A:L

Métrica Valor Sigla Razón Técnica
Vector de ataque Red AV:N Se explota remotamente sobre protocolo de red.
Complejidad Bajo AC:L No existen barreras de seguridad adicionales.
Privilegios req. Alto PR:H Requiere permisos de administrador.
Interacción Ninguno UI:N Puede ejecutarse sin ayuda de terceros.
Alcance Sin cambios S:U Impacto confinado al sistema web afectado.
Impacto CIA Bajo C:L/I:L/A:L Pérdida parcial en Confidencialidad, Integridad y Disponibilidad.

4. Interpretación Técnica

A pesar de que el requisito de privilegios elevados (PR:H) actúa como un control de seguridad inicial, la vulnerabilidad no pierde relevancia. Un atacante que logre comprometer una cuenta administrativa a través de otros vectores usaría este fallo para realizar movimientos laterales o alterar información sin interacción del usuario. El perfil típico es un insider o un actor externo persistente. Dado que el impacto es Bajo, la explotación no implica un desastre total, pero sí una degradación de la confianza en los datos de la aplicación.

5. Cálculo de Puntuación

Utilizando la calculadora oficial del FIRST (Forum of Incident Response and Security Teams):

  • Puntuación Base: 4.3
  • Clasificación de Severidad: Media (Rango 4.0 - 6.9)

6. Conclusión

La evaluación arroja un resultado de 4.3 (Severidad Media). Debe ser registrada en el inventario de seguridad y se recomienda su remediación después de que vulnerabilidades "Críticas" y "Altas" hayan sido mitigadas. La principal medida preventiva es el fortalecimiento del control de acceso y el monitoreo estricto de cuentas administrativas para evitar que esta brecha sea utilizada en ataques de mayor escala.
