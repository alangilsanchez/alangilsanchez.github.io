ACTIVIDAD 04: Mecanismos de defensa en red

Febrero 2026 Alan Gilberto Sánchez Zavala (177263)

Introducción contextual al tema

En la administración de redes seguras, es imperativo establecer políticas de control de tráfico que minimicen la superficie de ataque. Esta actividad se centra en la implementación práctica de políticas para un servidor multifuncional (Web, Correo y DNS), aplicando el principio de "concesión mínima de privilegios" mediante el uso de iptables. Se analizarán reglas para permitir conectividad selectiva tanto para tráfico entrante como saliente.

Desarrollo técnico de la actividad

A continuación se detallan las reglas de firewall implementadas para cumplir con los requerimientos específicos de seguridad de la red:

1. Establecer una política restrictiva

Se configura una regla base para aceptar tráfico TCP únicamente en los puertos web estándar (HTTP/HTTPS).

iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT

2. Permitir el tráfico de conexiones ya establecidas

Es vital permitir el tráfico de retorno para conexiones que ya han sido aprobadas (Stateful Firewalling).

iptables -A INPUT -p tcp -m state --state ESTABLISHED -j ACCEPT

3. Aceptar tráfico DNS (TCP) saliente de la red local

Permite que el servidor realice consultas DNS hacia el exterior.

iptables -A OUTPUT -p tcp -j ACCEPT

Nota: Generalmente DNS usa UDP (puerto 53), pero TCP se usa para transferencias de zona o respuestas grandes.

4. Aceptar correo entrante proveniente de Internet

Se permite tráfico dirigido específicamente al servidor de correo (IP interna 192.1.2.10).

iptables -A INPUT -p tcp -d 192.1.2.10 -j ACCEPT

5. Permitir correo saliente a Internet

Se habilita al servidor de correo (192.1.2.10) para enviar mensajes hacia el exterior.

iptables -A OUTPUT -p tcp -s 192.1.2.10 -j ACCEPT

6. Aceptar conexiones HTTP desde Internet al servidor web

Regla específica para permitir tráfico web entrante destinado a la IP del servidor web (192.1.2.11).

iptables -A INPUT -p tcp -s 192.1.2.11 --dport 80 -j ACCEPT

Correction Note: El flag -s indica origen (source). Para destino debería ser -d, o si la regla está en el servidor web mismo, solo --dport.

7. Permitir tráfico HTTP desde la red local a Internet

Permite que los usuarios internos naveguen por la web.

iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT

Opinión o reflexión técnica del estudiante

La implementación de políticas en iptables requiere una precisión absoluta. Un error tipográfico en una dirección IP o un puerto (como confundir source -s con destination -d) puede dejar un servicio inoperativo o, peor aún, expuesto. Esta práctica refuerza la importancia de entender el flujo del tráfico: saber qué entra (INPUT), qué sale (OUTPUT) y cómo el estado de las conexiones (stateful vs stateless) afecta la seguridad operativa de servicios críticos como correo y web.
