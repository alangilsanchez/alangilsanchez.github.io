ACTIVIDAD 06: Implementación IPSec VPN

Febrero 2026 Alan Gilberto Sánchez Zavala (177263)

Introducción

En esta actividad se implementó una VPN IPSec punto a punto entre dos redes locales, interconectadas a través de un router ISP. La finalidad es asegurar la comunicación entre las LAN 192.168.1.0/24 y 192.168.3.0/24 mediante cifrado AES-256 y autenticación con claves precompartidas.

Desarrollo técnico

Topología utilizada

R1 se conecta al ISP mediante la red 209.165.100.0/24 y R3 mediante la red 209.165.200.0/24. Cada router protege su red LAN interna usando un túnel IPSec.

Configuración de interfaces y rutas


R1(config)# interface g0/0
R1(config-if)# ip address 209.165.100.1 255.255.255.0
R1(config-if)# no shutdown
R1(config-if)# exit
R1(config)# interface g0/1
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
R1(config-if)# exit
R1(config)# ip route 0.0.0.0 0.0.0.0 209.165.100.2


R3(config)# interface g0/0
R3(config-if)# ip address 209.165.200.1 255.255.255.0
R3(config-if)# no shutdown
R3(config-if)# exit
R3(config)# interface g0/1
R3(config-if)# ip address 192.168.3.1 255.255.255.0
R3(config-if)# no shutdown
R3(config-if)# exit
R3(config)# ip route 0.0.0.0 0.0.0.0 209.165.200.2

Configuración del ISP


ISP(config)# interface g0/0
ISP(config-if)# ip address 209.165.100.2 255.255.255.0
ISP(config-if)# no shutdown
ISP(config-if)# exit
ISP(config)# interface g0/1
ISP(config-if)# ip address 209.165.200.2 255.255.255.0
ISP(config-if)# no shutdown

Lista de acceso (tráfico interesante)


R1(config)# access-list 100 permit ip 192.168.1.0 0.0.0.255 192.168.3.0 0.0.0.255
R3(config)# access-list 100 permit ip 192.168.3.0 0.0.0.255 192.168.1.0 0.0.0.255

ISAKMP, Transform-set y Crypto Map


R1(config)# crypto isakmp policy 10
R1(config-isakmp)# encryption aes 256
R1(config-isakmp)# authentication pre-share
R1(config-isakmp)# group 5
R1(config-isakmp)# exit
R1(config)# crypto isakmp key secretkey address 209.165.200.1
R1(config)# crypto ipsec transform-set R1-R3 esp-aes 256 esp-sha-hmac
R1(config)# crypto map IPSEC-MAP 10 ipsec-isakmp
R1(config-crypto-map)# set peer 209.165.200.1
R1(config-crypto-map)# set pfs group5
R1(config-crypto-map)# set security-association lifetime seconds 86400
R1(config-crypto-map)# set transform-set R1-R3
R1(config-crypto-map)# match address 100
R1(config)# interface g0/0
R1(config-if)# crypto map IPSEC-MAP


R3(config)# crypto isakmp policy 10
R3(config-isakmp)# encryption aes 256
R3(config-isakmp)# group 5
R3(config-isakmp)# exit
R3(config)# crypto isakmp key secretkey address 209.165.100.1
R3(config)# crypto ipsec transform-set R3-R1 esp-aes 256 esp-sha-hmac
R3(config)# crypto map IPSEC-MAP 10 ipsec-isakmp
R3(config-crypto-map)# set peer 209.165.100.1
R3(config-crypto-map)# set pfs group5
R3(config-crypto-map)# set security-association lifetime seconds 86400
R3(config-crypto-map)# set transform-set R3-R1
R3(config-crypto-map)# match address 100
R3(config)# interface g0/0
R3(config-if)# crypto map IPSEC-MAP

Prueba de conectividad

Se configuraron las PCs con sus respectivas direcciones IP, máscara y gateway. Posteriormente se realizó un ping desde la red 192.168.3.0 hacia la dirección 192.168.1.10, verificando el establecimiento exitoso del túnel IPSec.

Conclusión

La práctica demuestra que IPSec es una solución efectiva para asegurar comunicaciones entre redes remotas. La correcta configuración de políticas ISAKMP, transform-sets y ACLs es esencial para garantizar la estabilidad y seguridad del túnel VPN.
